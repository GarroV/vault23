# Task Assistant Bot

AI-помощник для ведения задач в Telegram. Мультитенантный SaaS.
Стек: Supabase (Postgres + pgvector + Edge Functions + Storage) · TypeScript/Deno · Telegram Bot API · OpenAI.

---

## С чего начать (в новой сессии Claude Code)

1. Claude Code сам прочитает `CLAUDE.md` — это рабочие правила проекта.
2. Для полного контекста скажи: «прочитай `MASTER_PROJECT.md`» — там всё: концепция, ТЗ, план, решения, открытые вопросы.
3. Текущее место в разработке — в `BUILD_PLAN.md`, раздел «ТЕКУЩАЯ ТОЧКА».

## Карта документов

**Источники правды (справочники):**
- `MASTER_PROJECT.md` — полный единый контекст проекта (главный документ)
- `PROJECT_OVERVIEW.md` — концепция, развёрнуто
- `BUILD_PLAN.md` — пошаговый рабочий чек-лист + эксплуатационная готовность
- `DEV_PLAN.md` — распределение блоков между Claude и Gemini
- `BACKLOG.md` — живой беклог (хочухи, техдолг, отложенное)

**Рабочие правила:**
- `CLAUDE.md` — всегда-действующие правила (читается автоматически)

**Пакет контекста (заполняется на этапе 1, до генерации кода):**
- `DATA_MODEL.md` — таблицы, поля, связи, RLS
- `MODULE_CONTRACT.md` — как модуль общается с ядром
- `CONVENTIONS.md` — именование, формат ошибок, стиль

**Проектные (по мере приближения этапов):**
- `BILLING_AND_ACCESS_DESIGN.md` — платежи и доступы (до Этапа 9)

## Структура репозитория (план)

```
/                        — документация (этот корень)
/functions               — Supabase Edge Functions (код)
/supabase/migrations     — версионированные миграции БД (supabase CLI)
/supabase/config.toml    — конфиг Supabase CLI
/docs                    — дополнительная документация
/decisions               — журнал архитектурных решений
```

## Порядок разработки (кратко)

Этап 0 окружение → 1 пакет контекста → 2 схема+RLS → 3 ядро → 4 задачи → 5 заметки/файлы/голос → 6 напоминания/лимиты → 7 база знаний → 8 календарь/сложное → 9 коммерциализация.

Подробности и пометки (что кому) — в `BUILD_PLAN.md` и `DEV_PLAN.md`.

## Ключевые принципы (полностью — в CLAUDE.md)
Мультитенантность + RLS · логика не в боте · модульность · review-gate · i18n РУ+АНГ · время в UTC · идемпотентность · Gemini только для рутины.
