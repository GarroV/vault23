## 9.1 — Платёжный провайдер

**Решение: Stripe**

Обоснование:
- Поддерживает 130+ валют, подписки, пробные периоды нативно.
- Webhook-надёжность: гарантирует retry с экспоненциальной задержкой. Критично для `invoice.payment_failed`.
- Stripe Tax — автоматический расчёт и сбор EU VAT без нашей инфраструктуры.
- PCI DSS из коробки: карты никогда не трогаем (закрывает требование 9.6 автоматически).
- Богатая документация + известные паттерны интеграции с Supabase Edge Functions.

Что делает Stripe вместо нас:
- Хранение карт (Stripe Vault).
- Расчёт и уплата НДС (EU, UK, AU...) через Stripe Tax.
- 3D Secure / SCA.
- Детектирование мошенничества (Stripe Radar).

Что делаем мы:
- Stripe Customer per workspace (создаём при первой оплате).
- Subscription per workspace (не per user).
- Обрабатываем вебхуки и обновляем статус воркспейса.
- Показываем пользователю checkout ссылку из бота.

Ограничения:
- Stripe недоступен напрямую для резидентов РФ/BY/ряда стран — клиенты из этих стран должны использовать промежуточные способы оплаты или другого провайдера. На старте игнорируем, в будущем — Paddle как альтернатива для СНГ.

## 9.2 — Матрица тарифов

### Тарифы

| | **Trial** (14 дней) | **Solo** ($9/мес) | **Team** ($29/мес) |
|---|---|---|---|
| Задачи | 20 | ∞ | ∞ |
| Подзадачи | ❌ | ✅ | ✅ |
| Заметки/встречи | ∞ | ∞ | ∞ |
| База знаний (записей) | 5 | 100 | 500 |
| Голос (распознаваний/мес) | 3 | 50 | 200 |
| Email-отправка/мес | ❌ | 20 | 100 |
| Google Calendar | ❌ | ✅ | ✅ |
| Вложения | 5 файлов × 5 МБ | 50 файлов × 20 МБ | 200 файлов × 50 МБ |
| Участников воркспейса | 1 (только owner) | 1 | 5 |

### Ключевые решения

**Триал — без карты.** Ниже барьер входа. Защита от повторных триалов: при истечении — блокировка (не урезанный free tier). Идентификатор для антифрода: Telegram ID. Повторный триал с тем же Telegram ID — не разрешается (workspace_id+telegram_id в anti_trial_log).

**В конце триала — полная блокировка (не free tier).** Никакого вечного бесплатного пользования. Пользователь видит "Trial ended, please subscribe" + ссылку на оплату. Данные хранятся 30 дней, потом воркспейс удаляется.

**Grace период — 7 дней после failed payment.** Не рубим сразу: карта могла протухнуть. В grace — работает, но каждый запрос получает уведомление "оплата не прошла, обнови карту".

**Downgrade (Solo → Trial/nothing) — только отмена.** Нет free tier, который сохраняет задачи вечно. При отмене — grace 30 дней, потом данные удаляются (с предупреждением).

**Лимиты — по двум осям:**
- Тарифный лимит (хранится в конфиге плана).
- Защитный потолок (мягкий, поднимается вручную через admin-команду).

## 9.3 — Роли внутри воркспейса

### Роли

| Роль | Кто | Что может |
|---|---|---|
| `owner` | Создатель воркспейса | Всё + управление биллингом + удаление воркспейса |
| `admin` | Назначен owner | Управление участниками + все фичи, кроме биллинга |
| `member` | Обычный участник | Все фичи по тарифу, нет управления workspace |

На пилоте: только `owner`. `admin` и `member` — когда Team-тариф станет активен.

### Таблица `workspace_members`

Хранит кто в каком воркспейсе и с какой ролью.
На пилоте: одна запись (owner) на воркспейс.
При мультиворкспейсе: мигрировать данные из `users.workspace_id`.

## 9.4 — Единая точка проверки доступа (Gate)

### Принцип

Одна функция `gate(feature, context)` вызывается в каждом обработчике перед выполнением действия.
- Проверяет статус воркспейса.
- Проверяет, доступна ли фича по тарифу.
- Проверяет count-лимиты (через DB-запрос).

### Статусы воркспейса

| Статус | Что значит | Доступ к фичам |
|---|---|---|
| `trial` | Идёт триал | По триальным лимитам |
| `active` | Подписка действует | По тарифным лимитам |
| `past_due` | Оплата не прошла, grace период | Работает + предупреждение |
| `suspended` | Grace истёк | Заблокирован |
| `cancelled` | Отменён самим клиентом | Заблокирован |

### Фичи (feature strings)

```
'task_create'     — создание задачи
'subtask_create'  — создание подзадачи
'voice'           — голосовое распознавание
'calendar'        — Google Calendar sync
'email_send'      — отправка email
'kb_add'          — добавление в базу знаний
'attachment'      — загрузка вложений
```

### Конфиг планов (TypeScript, не в БД)

```typescript
const PLAN_LIMITS = {
  trial:  { maxTasks: 20, maxKb: 5, maxVoiceMonth: 3, maxEmailMonth: 0, calendar: false, subtasks: false },
  solo:   { maxTasks: Infinity, maxKb: 100, maxVoiceMonth: 50, maxEmailMonth: 20, calendar: true, subtasks: true },
  team:   { maxTasks: Infinity, maxKb: 500, maxVoiceMonth: 200, maxEmailMonth: 100, calendar: true, subtasks: true },
}
```

## 9.5 — Обработка сбоев оплаты

| Событие Stripe | Наш ответ |
|---|---|
| `checkout.session.completed` | Активировать воркспейс, сохранить customer_id + subscription_id |
| `invoice.payment_succeeded` | Продлить period_end, если статус не `active` — вернуть в `active` |
| `invoice.payment_failed` | Если `active` → `past_due`; уведомить пользователя в Telegram |
| `customer.subscription.deleted` | Статус → `cancelled`; уведомить |
| `customer.subscription.updated` | Обновить план, если изменился `price_id` |

### Идемпотентность вебхуков

Stripe может прислать один и тот же вебхук дважды. Обрабатываем через проверку `stripe_event_id` в отдельной таблице `processed_stripe_events`.

## 9.6 — Карты не храним

Закрывается автоматически через Stripe:
- Checkout Session — пользователь вводит данные карты на странице Stripe.
- Данные карты к нам никогда не приходят.
- Payment Method хранится в Stripe Vault под customer_id.
- Мы храним только `stripe_customer_id` и `stripe_subscription_id` — это публичные идентификаторы, не чувствительные данные.

## 9.7 — Управление подпиской

| Действие | Механика |
|---|---|
| Апгрейд (Solo → Team) | Создать новый Stripe Billing Portal Session → пользователь меняет план сам |
| Даунгрейд (Team → Solo) | То же через Billing Portal; Stripe считает proration |
| Отмена | Через Billing Portal; подписка заканчивается в конце оплаченного периода |
| Возврат данных | При отмене данные живут grace 30 дней, потом удаляются |

Billing Portal — делегируем Stripe. Не писать собственный UI управления подпиской.
