# CONVENTIONS.md — соглашения разработки

> Именование, формат ошибок, стиль, структура модуля. Заполняется на шаге 1.3 (🔴 Claude).
> Чтобы код (в т.ч. сгенерированный Gemini) был единообразным.
> Статус: ЗАГОТОВКА — ключевые правила зафиксированы, детали стиля дополняются при сборке.

## i18n (зафиксировано)
- Все интерфейсные тексты — через ключи, никогда не хардкодом.
- Языковые файлы `ru` и `en`. Оба заполняются СРАЗУ при разработке каждого блока, не «английский потом».
- Ключи именуются по смыслу: `task_created`, `error_file_too_large` и т.п.
- Автоопределение языка: `language_code` ru→русский, иначе→английский (дефолт/фолбэк).
- Ручная смена языка + сохранение в профиль.
- Данные пользователя (названия задач, заметки, темы) НЕ переводятся.

## Стратегия ошибок (зафиксировано)
- Бот никогда не молчит и не падает.
- Пользователю — только понятные человеческие сообщения. Техника (стек, детали) — в лог.
- Три класса:
  - *Ошибка пользователя* (большой файл, пустой ввод) → спокойно объяснить, что не так и что сделать.
  - *Временная системная* (внешний сервис лёг) → «попробуй через минуту» + авто-retry пару раз перед сдачей.
  - *Баг* → извиниться + детальная запись в лог.
- Не терять пользовательский ввод: сначала сохранить сырое, потом обрабатывать.
- В логах — без чувствительного содержимого заметок открытым текстом.
- Не усложнять: мониторинг/алертинг — для большой нагрузки, не для пилота.

## Миграции (зафиксировано)
- Изменения схемы на живых данных — только версионированной миграцией (Supabase CLI), не руками в прод.
- У каждой миграции: понятное имя, возможность отката, дефолты для существующих строк.
- На живых данных — сначала проверить на копии, потом применять.
- Не удалять колонки/таблицы с данными без явного бэкапа.

## Именование (намечено)
- Таблицы — множественное, snake_case.
- Поля — snake_case.
- Функции/переменные в коде — по стандарту TypeScript (camelCase), детализируется при сборке.

## Структура модуля (намечено)
Единый каркас файла модуля по `MODULE_CONTRACT.md`: регистрация, обработчики команд, доступ к контексту. Детализируется при сборке ядра.

## Открыто к детализации при сборке
Точный стиль кода (форматтер/линтер), структура папок внутри `/functions`, шаблон нового модуля, конвенция логирования.
